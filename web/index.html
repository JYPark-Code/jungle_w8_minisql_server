04 · Tree Anatomy

B+ Tree 해부.

레코드 수 (N) — 20 5 ~ 200, 드래그하면 실시간 재생성 (release 시)
분기 수 (Order)
휠 = 확대 / 드래그 = 이동
레코드 수 (N) — 트리에 삽입할 id 개수. 1..N 을 랜덤 순서로 넣어 실제 split 이 발생하는 모습을 보여준다. 최대 200 (화면 폭 제한).
분기 수 (Order) — 한 노드가 가질 수 있는 최대 자식 개수. 작을수록 (4) 트리가 금방 깊어져 split 구조가 잘 보이고, 클수록 (128) 실제 프로덕션처럼 납작해진다.
위 트리 그리기 또는 삽입 애니메이션 을 눌러주세요.
이 트리가 말해주는 것
  1. 깊이 = O(log n) — 루트에서 리프까지 3~4 단계. 레코드가 100만 건이어도 단계가 거의 안 늘어난다. 이게 SEARCH 2.18M ops/s 의 근거.
  2. 리프 linked list (빨간 점선) — 범위 조회가 트리 깊이를 다시 내려가지 않고 리프끼리 링크를 따라가며 O(log n + k) 로 끝난다.
  3. split 발생 지점 — 노드가 꽉 차면 가운데 키가 부모로 올라가고 자식이 둘로 나뉜다. 키가 정렬된 채로 분기가 유지되는 이유.
  4. Order = 4 — 키 N-1 개, 자식 N 개 분기. 프로덕션에선 128~256 으로 납작하게 유지.
